# An Introduction to Medical Astrology (Wanda Sellar) — Çok-Yöneticili Korelasyon + Dignite Katmanı

> **Eser:** *An Introduction to Medical Astrology* — Wanda Sellar
> **Modül:** 🌿 Herbalizm/medikal-astroloji kasası + ✨ kişiselleştirme (dignite mantığı). Modern, akademik ders kitabı.
> **Öne çıkan:** Sellar'ın kendisi **bizim güvenlik çizgimizi savunuyor** — "hastalık adı vermek iyi fikir değil; telkin güçlü bir araçtır; teşhis tıbbi personele aittir." Bu, app'in teşhis-dışı yaklaşımını **birincil kaynaktan doğrular.**
> **⚠️ Sınır:** Decumbiture (hastalıktan teşhis), ameliyat significatorları (tıbbi zamanlama) ve hastalık adlandırma app'e **girmez.**

## 1. ⭐ Sellar'ın güvenlik duruşu (app çizgisini doğrular)

Sellar, genel okumada hastalık yatkınlığı söylemenin tehlikesini açıkça belirtir: telkin güçlüdür, kişi hastalığı geliştirmese bile umutsuzluğa düşebilir; organ işlev bozukluğu sayısız hastalığın sonucu olabilir, **teşhis nihayetinde ortodoks tıbba aittir.**

> App için: Bu, "teşhis/etiket yapma, nazik öz-bakıma çevir" kuralımızın **alanın kendi otoritelerince** desteklendiğinin kanıtı — farklılaştırıcı güvenlik duruşunu güçlendirir.

## 2. ⭐ Çok-yöneticili beden korelasyonu (zenginleştirme)

Sellar, çoğu beden bölümünün **birden çok yöneticisi** olduğunu vurgular — tek-eşlemeli melothesia'dan daha nüanslı.

| Beden bölümü | Yöneticiler |
|---|---|
| Adrenaller | Mars, Jüpiter, Terazi, Akrep |
| Sinir sistemi | Merkür (geleneksel) + Uranüs (kimyasal-elektriksel) |
| Beyin | Koç (Güneş burada yücelir) |
| Bağırsaklar | Başak |
| Bademcikler | Boğa, Venüs |
| Karaciğer | Jüpiter |
| Arteriyel sistem (iliak/safen) | Venüs/Jüpiter bağlantılı |

> Tasarım notu: app'in beden-korelasyon katmanı tek gezegen yerine **çoklu yönetici** tutabilir → daha zengin, "birden çok testimony" mantığı.

## 3. Element → gezegen + konstitüsyon → dosha

Sellar, Ayurveda köprüsünü açık verir (Popham'la **tutarlı**):

| Element | Gezegen | | Konstitüsyon | Yöneticiler |
|---|---|---|---|---|
| Ether | Jüpiter | | **Vata** | Toprak burçları + Satürn, Kuzey Düğüm, Merkür |
| Hava | Satürn | | **Pitta** | Ateş burçları + Güneş, Mars, Güney Düğüm |
| Ateş | Güneş, Mars | | **Kapha** | Su burçları + Ay, Venüs, Jüpiter |
| Su | Ay, Venüs | | | |
| Toprak | Merkür | | | |

## 4. Gezegen → bitki ilkesi + beden lateralitesi

- **Benzer nitelik ilkesi:** Mars (sıcak-kuru) → sıcak-kuru bitkiler (zencefil). Venüs bitkileri → doğurganlık desteği; Mars bitkileri → güçlendirme; Ay bitkileri → diüretik. (Kasadaki sempati ilkesiyle tutarlı.)
- **Lateralite:** Güneş bedenin **sağ** yanını, Ay **sol** yanını yönetir. (App'in beden-haritası içeriğine ince bir katman.)
- ⚠ **Toksik not:** Sellar tarihsel "civa merhemi" (cilt/uyuz/zührevi) örneğini verir — **app dışı**, civa toksik.

## 5. Dignite / debilite (kişiselleştirme mantığı)

Sellar, gezegen gücünü **esansiyel dignitelerle** ölçer: domicile/exaltation (güçlü, iyi işleyen) ↔ detriment/fall (çarpık enerji, "debility"). Zor yerleşim ≠ kötü; **psikolojik bakışla "zorluk/davet"** olarak çerçevelenir.

> App için: kişiselleştirme algoritması bir gezegenin gücünü (dignity) okuyarak içeriğin **tonunu** seçebilir — ama "debility = hastalık" demez; nazik "bu alanda destek/denge" diline çevrilir.
> (Not: OCR'daki tam dignite tablosu bozuk çıktı; standart domicile değerleri Daath raporunda temiz mevcut — orayı referans al.)
